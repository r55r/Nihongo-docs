## 目的

N2の新規問題と弱点復習を混ぜ、10分で完了できる毎日の演習を提供する。

## 対象ユーザー

- 対象レベル: JLPT N2
- 利用前提: 学習計画が作成済み

## ユーザーフロー

1. HomeまたはLearnから演習を開始する。
2. 1問ずつ回答する。
3. 正誤、短い解説、必要に応じたAI補足を表示する。
4. 完了時に学習履歴と弱点候補を更新する。

## 画面/状態

| 画面または状態 | 主アクション | 表示内容 | 遷移先 |
| --- | --- | --- | --- |
| 問題 | `回答する` | 問題文、選択肢、進捗 | 解説 |
| 解説 | `次へ` | 正誤、解説、AI補足 | 次の問題 |
| 完了 | `Homeへ` | 完了、streak、次の提案 | Home |

含める状態: ローディング、空状態、成功、エラー、オフライン、権限不足。

## データ要件

| データ | 型/形式 | 必須 | 説明 |
| --- | --- | --- | --- |
| sessionId | string | yes | 演習セッションID |
| questionId | string | yes | 問題ID |
| answer | string | yes | 選択肢ID |
| isCorrect | boolean | yes | 正誤 |
| confidence | `low` / `mid` / `high` | no | 自信度 |
| elapsedMs | number | yes | 回答時間 |

## API/Firebase 要件

React Query key は `["practiceSession", sessionId]`。AI補足は Functions `answerExplain` と共有キャッシュを使う。

## コンテンツ要件

- N2問題のみ出題する。
- 問題にはレビュー状態と公開状態を持たせる。
- 未許諾本文を問題として使わない。

## エッジケース

- 未ログイン: ゲスト保存。
- データ未作成: Homeへ戻す。
- 通信失敗: AI補足なしで解説を表示。
- 途中離脱: セッションを再開可能にする。
- 重複送信: 同じ問題の回答は1回だけ保存。
- 端末変更: 初回リリースでは引き継がない。

## 実装対象外

- 聴解問題。
- 記述式採点。
- N2以外の演習。

## 受け入れ条件

- [ ] 新規問題と復習問題が混ざる。
- [ ] 回答後に正誤と解説が出る。
- [ ] 結果が弱点候補に反映される。
