# N2毎日学習計画 下書き

| --- | --- |
| updated | 2026-05-26 |

## 目的

10問診断と試験予定日から、毎日迷わず進められるN2学習計画を作る。

## 対象ユーザー

- 対象レベル: JLPT N2
- 利用前提: オンボーディングと10問診断を完了している

## ユーザーフロー

1. 診断結果から初期計画を作る。
2. Homeに今日のおすすめとして表示する。
3. 演習、復習、ミニ模試の結果で計画を更新する。

## 画面/状態

| 画面または状態 | 主アクション | 表示内容 | 遷移先 |
| --- | --- | --- | --- |
| 今日の計画 | `始める` | 10分目安、対象大問、進捗 | 演習 |
| 完了 | `復習する` | streak、次の提案 | 復習 |
| 未作成 | `診断へ` | 計画がない説明 | 診断 |

含める状態: ローディング、空状態、成功、エラー、オフライン、権限不足。

## データ要件

| データ | 型/形式 | 必須 | 説明 |
| --- | --- | --- | --- |
| planId | string | yes | 学習計画ID |
| dailyStudyMinutes | number | yes | 1日の学習時間 |
| targetDate | ISO date | yes | 試験予定日 |
| tasks | object[] | yes | 今日のタスク |
| completedTaskIds | string[] | yes | 完了済み |

## API/Firebase 要件

React Query key は `["studyPlan", guestId]`。初回リリースはローカル保存、後続同期先は `users/{userId}/studyPlans/{planId}`。

## コンテンツ要件

タスクはN2大問タグに紐づける。N2以外のタスクは生成しない。

## エッジケース

- 未ログイン: ゲスト保存。
- データ未作成: 診断へ戻す。
- 通信失敗: ローカル計画で継続。
- 途中離脱: 未完了タスクを保持。
- 重複送信: 完了処理を冪等にする。
- 端末変更: 初回リリースでは引き継がない。

## 実装対象外

- N2以外の計画。
- カレンダー連携。
- 進学・就職ゴール連携。

## 受け入れ条件

- [ ] 診断後に初期計画が作られる。
- [ ] Homeに今日のおすすめが出る。
- [ ] N2以外のタスクが出ない。
