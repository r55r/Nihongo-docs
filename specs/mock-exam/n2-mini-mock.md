## 目的

短時間でN2本番に近い確認を行い、弱点と合格可能性を更新する。

## 対象ユーザー

- 対象レベル: JLPT N2
- 利用前提: 毎日の演習または弱点復習を一定量完了している

## ユーザーフロー

1. HomeまたはLearnからミニ模試を開始する。
2. 10〜20問を連続で解く。
3. 完了後に分野別結果と弱点候補を表示する。
4. 合格可能性を更新する。

## 画面/状態

| 画面または状態 | 主アクション | 表示内容 | 遷移先 |
| --- | --- | --- | --- |
| 開始 | `始める` | 問題数、所要時間、対象分野 | 問題 |
| 問題 | `次へ` | 問題、進捗 | 次の問題 |
| 結果 | `復習する` | スコア、分野別結果、次の提案 | 弱点復習 |

含める状態: ローディング、空状態、成功、エラー、オフライン、権限不足。

## データ要件

| データ | 型/形式 | 必須 | 説明 |
| --- | --- | --- | --- |
| mockExamId | string | yes | 模試ID |
| questionIds | string[] | yes | 10〜20問 |
| score | number | yes | 正答数または正答率 |
| sectionScores | object | yes | 大問タグ別結果 |
| completedAt | ISO datetime | yes | 完了日時 |

Zod では問題数を10〜20問に制限し、N2問題だけで構成する。

## API/Firebase 要件

React Query key は `["mockExam", mockExamId]`。後続同期先は `users/{userId}/mockExams/{mockExamId}`。

## コンテンツ要件

- N2の文字・語彙、文法、読解から構成する。
- 聴解は含めない。
- 公式問題はコピーしない。

## エッジケース

- 未ログイン: ゲスト保存。
- データ未作成: 模試準備中を表示。
- 通信失敗: 同梱問題で開始可能。
- 途中離脱: 未完了模試として再開または破棄を選べる。
- 重複送信: 完了保存を1回にする。
- 端末変更: 初回リリースでは引き継がない。

## 実装対象外

- フル模試。
- 聴解。
- 厳密な本番時間再現。

- [ ] 分野別結果が表示される。
- [ ] 結果が弱点と合格可能性に反映される。
