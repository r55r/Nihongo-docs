# 3. 診断テスト機能

## 3.1 診断テストの基本仕様

| 項目   | 内容                 |
| ---- | ------------------ |
| 所要時間 | 8〜12分              |
| 問題数  | 12〜20問             |
| 対象   | N3/N2判定用           |
| 出題範囲 | 語彙、文法、読解、聴解        |
| 出題方式 | 初期はルールベースの簡易アダプティブ |
| 結果保存 | 初回診断結果として保存        |
| 再受験  | 7日ごとに再診断可能         |

## 3.2 診断テストの出題ロジック

| 機能           | 内容                              |
| ------------ | ------------------------------- |
| 目標レベル別出題     | N3志望者にはN4〜N3、N2志望者にはN3〜N2を中心に出題 |
| 正答状況による難易度調整 | 連続正解なら難度を上げる、連続誤答なら下げる          |
| スキル別配分       | 語彙25%、文法25%、読解25%、聴解25%         |
| 回答時間記録       | 正誤だけでなく回答速度も保存                  |
| 自信度入力        | 任意で「自信あり / 迷った / 勘」を選択可能        |

## 3.3 診断結果画面

| 表示        | 内容                  |
| --------- | ------------------- |
| 総合準備度     | 0〜100%              |
| 推定JLPTレベル | N3準備中、N3合格圏、N2挑戦圏など |
| スキル別スコア   | 語彙、文法、読解、聴解         |
| 弱点トップ3    | 例：助詞、長文読解、聴解の要点把握   |
| 合格可能性     | 目標レベルに対する推定         |
| 推奨学習時間    | 試験日までの必要学習時間        |
| 今日のメニュー   | そのまま学習開始できる         |

# 4. 合格可能性・準備度推定機能

## 4.1 スコア推定

| 機能      | 内容                 |
| ------- | ------------------ |
| 総合準備度   | 0〜100で表示           |
| スキル別準備度 | 語彙、文法、読解、聴解別       |
| レベル別準備度 | N3準備度、N2準備度        |
| 信頼度表示   | データ不足時は「推定精度：低」と表示 |
| 推定更新    | 診断、練習、模試の結果で毎日更新   |

## 4.2 合格可能性表示

| 機能           | 内容                       |
| ------------ | ------------------------ |
| 合格可能性パーセンテージ | 例：N2合格可能性 48%            |
| 注意文言         | 「実際の合格を保証するものではありません」と表示 |
| 推定根拠         | 語彙、文法、読解、聴解のどこが影響しているか表示 |
| 改善シミュレーション   | 「7日間続けると推定+4%」など         |
| 危険アラート       | 「読解速度が合格ラインより遅い」など       |

## 4.3 推定アルゴリズム v1

MVPでは高度な機械学習ではなく、以下のルールベースで十分です。

| 入力     | 使い方            |
| ------ | -------------- |
| 正答率    | スキル別の理解度推定     |
| 回答時間   | 時間内に解けるかを判定    |
| 問題難易度  | 難問正解には高い重みを付ける |
| 誤答タイプ  | 弱点カテゴリに反映      |
| 学習継続日数 | 短期的な定着度に反映     |
| 復習成功率  | 忘却リスクの補正       |
| 模試結果   | 最も重い評価データとして反映 |

## 9.2 AI解説

OpenRouterを使ってください。
DeepSeek V4 Flashを使ってください。

| --------- | -------------------- |
| 個別誤答解説    | ユーザーが選んだ選択肢に応じた説明    |

# 10. 誤答原因分類機能

## 10.1 誤答タイプ

| カテゴリ    | 内容                 |
| ------- | ------------------ |
| 語彙不足    | 単語の意味を知らなかった       |
| 漢字読み    | 読みを間違えた            |
| 文脈判断    | 文の流れに合う語を選べなかった    |
| 文法混同    | 似た文法と混同した          |
| 活用ミス    | 動詞・形容詞の形を誤解した      |
| 助詞ミス    | に、で、を、が、は等の誤り      |
| 読解構造    | 逆接、理由、結論を見落とした     |
| 指示語ミス   | これ、それ、このような等を読み違えた |
| 筆者主張ミス  | 筆者の意見を取り違えた        |
| 情報検索ミス  | 必要情報を探せなかった        |
| 聴解聞き逃し  | 重要語を聞き逃した          |
| 聴解保持ミス  | 聞いた情報を覚えていられなかった   |
| 選択肢トラップ | 強すぎる表現、逆の意味に引っかかった |
| 時間不足    | 正答可能だが時間切れ         |
| ケアレスミス  | 理解していたが選択ミス        |

## 10.2 分類方法

| 方法      | 内容                     |
| ------- | ---------------------- |
| 問題タグ    | 各問題に想定誤答原因を事前設定        |
| 誤答選択肢タグ | 選択肢ごとに誤答理由を付与          |
| 回答時間    | 長すぎる場合は迷い・時間不足を検出      |
| 自己申告    | 回答後に「知らなかった」「迷った」などを選択 |
| AI補助    | 解答パターンから推定             |
| 履歴補正    | 同じカテゴリの連続誤答を重視         |

## 10.3 誤答後の動作

| 機能         | 内容         |
| ---------- | ---------- |
| 誤答ノート登録    | 自動で保存      |
| 復習スケジュール追加 | SRSに登録     |
| 関連問題推薦     | 同じ弱点の問題を出す |
| 弱点スコア更新    | ダッシュボードに反映 |

## 12.2 模試タイプ

| ------ | ----------- |

# 28. MVP v1で入れない機能

ここを明確に切ることが重要です。

| 入れない機能        | 理由              |
| ------------- | --------------- |
| EJU対策         | v1では焦点がぼやける     |
| JLPT N1/N4/N5 | コンテンツ範囲が広がりすぎる  |
| 完全な辞書機能       | Mazii等と正面衝突するため |
| OCR           | MVPの主価値ではない     |
| 手書き漢字認識       | 工数に対して合格価値が低い   |
| AI会話練習        | JLPT合格との直接性が弱い  |
| 発音採点          | JLPTでは優先度が低い    |
| ライブ授業         | オペレーションが重い      |
| 講師マッチング       | 別事業になる          |
| 学校向け管理画面      | v3以降            |
| フルサイズ模試大量搭載   | v1ではミニ模試中心      |
| SNS・コミュニティ    | 学習コア完成後でよい      |
| ランキング         | 初期は不要           |
| アバター・ゲーム要素過多  | 試験対策価値を優先       |
| 公式過去問掲載       | 権利処理なしでは不可      |

# 29. MVP v1の成功条件

| 指標            |                 目標 |

| AI解説利用率       |              30%以上 |

| 有料ユーザーの週次学習率  |              50%以上 |
| 問題報告率         | 低いほど良いが、初期は品質改善に活用 |
| 「今日やることが明確」評価 |         ユーザー調査で高評価 |
